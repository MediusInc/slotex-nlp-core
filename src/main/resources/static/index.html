1. Introduction

This is a Spring REST Docs documentation four our MediusNLP RESTfull API.

2. Admin API

Collection of CRUD API endpoints used to manipulate and show administration data about out MediusNLP platform.

2.1. GET status

Obtains a status message from servcer which tells us basic information about the processes that are running on our platform. It provides data about how many documents are already processed, in processing and waiting to be processed.

2.1.1. Sample Request

GET /admin/status HTTP/1.1
Host: localhost:8100

2.1.2. Sample Response

HTTP/1.1 200 OK
Content-Type: application/json;charset=UTF-8
Content-Length: 107

{
  "waitQueueSize" : 0,
  "workQueueSize" : 0,
  "waitDocumentIds" : [ ],
  "workDocumentIds" : [ ]
}
Response fields
Path Type Description

waitQueueSize

Number

Number of documents in the wait queue.

workQueueSize

Number

Number of documents in the work queue.

waitDocumentIds

Array

List of document IDs that are currently in the wait queue.

workDocumentIds

Array

List of document IDs that are currently in the work queue.

2.1.3. CURL sample

$ curl 'http://localhost:8100/admin/status' -i -X GET

2.2. POST document to queue

Pushes a document to the waiting to be processed queue.

2.2.1. Sample Request

POST /admin/push HTTP/1.1
Host: localhost:8100
Content-Length: 507
Content-Type: application/json;charset=UTF-8

{
  "documentId" : "3212",
  "insertDate" : null,
  "title" : "Nobelovi nagrajenci",
  "content" : "Niz razglasitev letošnjih prejemnikov Nobelovih nagrad so začeli v ponedeljek z nagrado za medicino, ki jo letos za imunoterapijo za zdravljenje raka prejmeta ameriški in japonski imunolog James P. Allison in Tasuku Honjo. V sredo bo znan nagrajenec za kemijo, v petek pa bodo razglasili prejemnika nagrade za mir. Dobitnik nagrade za dosežke na področju ekonomije bo znan naslednji ponedeljek."
}
Request fields
Path Type Description

documentId

String

Number ID of the document.

insertDate

Null

Date of the insertion to the queue. If not provided it defines the current one.

title

String

Title of the document to process

content

String

Content of the document that has to be processed and tagged.

2.2.2. Sample Response

HTTP/1.1 201 Created

2.2.3. CURL sample

$ curl 'http://localhost:8100/admin/push' -i -X POST \
    -H 'Content-Type: application/json;charset=UTF-8' \
    -d '{
  "documentId" : "3212",
  "insertDate" : null,
  "title" : "Nobelovi nagrajenci",
  "content" : "Niz razglasitev letošnjih prejemnikov Nobelovih nagrad so začeli v ponedeljek z nagrado za medicino, ki jo letos za imunoterapijo za zdravljenje raka prejmeta ameriški in japonski imunolog James P. Allison in Tasuku Honjo. V sredo bo znan nagrajenec za kemijo, v petek pa bodo razglasili prejemnika nagrade za mir. Dobitnik nagrade za dosežke na področju ekonomije bo znan naslednji ponedeljek."
}'

2.3. GET process from queue

Processes the desired number of documents from the wait queue.

2.3.1. Sample Request

GET /admin/process/1 HTTP/1.1
Host: localhost:8100
Content-Type: application/json;charset=UTF-8

2.3.2. Sample Response

HTTP/1.1 201 Created
Content-Length: 170
Content-Type: application/json;charset=UTF-8

{
  "numOfProcessed" : 1,
  "processedIds" : [ "3212" ],
  "processStartTime" : "2019-05-21T18:56:57.913+0000",
  "processEndTime" : "2019-05-21T18:57:00.980+0000"
}
Response fields
Path Type Description

numOfProcessed

Number

Number of how many documents were processed.

processedIds

Array

List of document IDs that were processed in this batch.

processStartTime

String

Time when the processing of documents in the queue has started.

processEndTime

String

Time when the processing of documents in the queue has ended.

2.3.3. CURL sample

$ curl 'http://localhost:8100/admin/process/1' -i -X GET \
    -H 'Content-Type: application/json;charset=UTF-8'

2.4. GET return documents to process

Returns all the documents from the work queue back to wait queue. There they can be processed again.

2.4.1. Sample Request

GET /admin/return HTTP/1.1
Host: localhost:8100
Content-Type: application/json;charset=UTF-8

2.4.2. Sample Response

HTTP/1.1 201 Created

2.4.3. CURL sample

$ curl 'http://localhost:8100/admin/return' -i -X GET \
    -H 'Content-Type: application/json;charset=UTF-8'
